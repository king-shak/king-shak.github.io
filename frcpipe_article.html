Custom FRC Pipelines

Inspiration

Majority of my work on my FRC team revolved around computer vision. CV and its applications to robotics have always been very interesting to me. However, most of what ended up on our robot used a system called the Limelight. This is an all-in-one computer vision platform that has a camera and a small computer on board that runs the computer vision pipelines. It was a very convenient solution, but it was expensive and would randomly crash (even during matches) quite often. So I set out to build my own alternative to the Limelight, mainly because I felt I could make something better and I've always wanted to implement my own computer vision pipelines. And I failed. But, I did learn a lot of things and I am still very proud of the things I did manage to do, which is why I wanted to list this on my portfolio.

How it Works

The hardware itself took various different forms but was still fundamentally the same as a Limelight - a Raspberry Pi with a camera, with some high-powered LED's for picking up targets. It was very similar to the Limelight in terms of software as well, it didn't work as well but it worked in the same way as you would configure its settings via a web browser.
It's very hard to explain this without going into all the little details which would take a long time. Instead, I'll talk about some of the challenges, what I learned, and link the code below.

Challenges

A lot. Obviously there were issues getting the web interface to work and all the other components involved with this system but the biggest challenges were (1) making sure the code ran as fast as possible, and (2) getting the hardware together. The former was because I was making the computer vision pipelines entirely by myself, and they needed to be able to process a certain number of frames per second (FPS) to be giving a steady stream of data to the robot. As for the latter, the hardware was a very difficult, mainly because I has set myself up to go for the small, neat form factor as the Limelight.
Obviously this whole project was very difficult, but I did it because I was very interested in it, I ended up learning many new things and it was a lot of fun.

What I Learned

In terms of technical things, a lot. I learned about web servers, WebSockets, threading, multiprocessing, inter-process communication (IPC) via pipes, queues, stacks, and sockets, double buffering, and a lot of OpenCV - it's an amazing library although it can be a bit difficult to debug at times.
Another thing I learned was the need to plan out a project in order to get it done on time, and the importance of having a team of people not only to work on it but to continue my work once I leave the team - because that was the biggest loss to me. I still don't regret doing this project and making these mistakes because of all the new things I learned.
